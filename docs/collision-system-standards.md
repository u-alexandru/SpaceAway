# Collision System Standards

## Core Principle

Almost everything in SpaceAway moves: ships, players, planets, asteroids, debris. This means **static-only collision types (TriMesh) cannot be used for most objects**. The collision system is built around dynamic-compatible shapes.

## 3-Tier Collider System

### Tier 1: Primitives (Cheapest)
- Shapes: Box, Sphere, Capsule, Cylinder
- Use for: Player body, projectiles, simple triggers, point-of-interest volumes
- Performance: ~0.001ms per collision pair
- Dynamic: Yes

### Tier 2: Auto-Convex Decomposition (Default for complex objects)
- Shapes: Convex hull or VHACD compound (auto-generated from mesh)
- Use for: Ship hull sections, furniture, interactables, asteroids, debris
- Performance: ~0.01ms per collision pair (depends on hull count)
- Dynamic: Yes
- **This is the default for anything generated by sa_meshgen**
- Rapier3d's built-in VHACD: `ColliderBuilder::convex_decomposition(vertices, indices)`

### Tier 3: TriMesh (Static geometry only)
- Shapes: Exact triangle mesh
- Use for: Planet surfaces, space station docking bays, terrain
- Performance: ~0.05ms per collision pair
- Dynamic: **NO — static/fixed bodies only**
- Accurate to the triangle level

## Auto-Collider Generation

Every sa_meshgen Mesh can automatically produce a matching collider:

```rust
// Tier 1: Simple box from bounding box
fn auto_collider_box(mesh: &Mesh) -> Collider

// Tier 2: Convex decomposition (default)
fn auto_collider_convex(mesh: &Mesh) -> Collider

// Tier 3: Exact triangle mesh (static only)
fn auto_collider_trimesh(mesh: &Mesh) -> Collider
```

The caller chooses the tier. Default is Tier 2.

## Collision Groups

| Group | Bit | Contents |
|-------|-----|----------|
| SHIP_HULL | 0x0001 | Ship exterior hull (sensor, mass only) |
| SHIP_INTERIOR | 0x0002 | Walkable surfaces inside ships |
| PLAYER | 0x0004 | Player capsule |
| INTERACTABLE | 0x0008 | Sensor volumes for interactions |
| WORLD | 0x0010 | Planets, asteroids, space structures |
| PROJECTILE | 0x0020 | Bullets, missiles |

Interaction mask rules:
- PLAYER collides with: SHIP_INTERIOR, WORLD
- INTERACTABLE detected by: raycast only (sensor, no contact forces)
- SHIP_HULL: no collision with anything (sensor for mass/inertia only)
- PROJECTILE collides with: SHIP_HULL, WORLD, PLAYER

## Moving Reference Frames

The ship moves through space. The player walks inside the moving ship.

**Phase 5a (current):** Ship is stationary while player walks. Simple.

**Phase 5b+ (future):** Ship moves with player inside. Two approaches:
1. **Velocity inheritance:** Each frame, add the ship's velocity to the player's movement. The player moves relative to the ship, and the ship's movement carries them.
2. **Local physics space:** Interior colliders are children of the ship rigid body. Rapier handles the reference frame transform. This is the cleaner approach but requires all interior colliders to be children of the ship body.

## Interactable Detection

Interactables use **sensor colliders in the INTERACTABLE group**. Detection via raycast:
- Ray origin: camera position
- Ray direction: camera forward
- Filter: INTERACTABLE group only (exclude everything else)
- Max range: 2.0m
- Result: collider handle → lookup in interactable registry

The hull sensor (SHIP_HULL group) is explicitly excluded from interaction rays.
