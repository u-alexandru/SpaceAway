# Phase 5a: Ship Core — Design Spec

## Overview

Make the ship flyable. Player walks inside the ship, interacts with physical controls (levers, buttons), sits at the helm, and flies through space with Newtonian physics. All controls are interaction-based — no keyboard shortcuts for ship functions.

## Scope

- sa_ship crate: Ship entity, interactable system, helm control
- Interactable meshes generated with sa_meshgen
- Ship spawned in space (replaces ground plane scene)
- Player walks inside, interacts with objects, flies from the helm

## NOT in scope (future phases)

- Power grid, life support, survival (Phase 5b)
- Upgrades, progression
- Multiplayer
- Audio
- HUD/UI beyond basic screen displays

## Architecture

### New Crate: sa_ship

Game Logic layer. Depends on: sa_core, sa_math, sa_physics, sa_ecs, sa_input.

**Modules:**

- `interactable.rs` — Interactable component types, state management
- `interaction.rs` — Raycast system, click-and-drag handling, interaction state machine
- `ship.rs` — Ship entity (physics body, subsystem registry, state)
- `helm.rs` — Helm seated mode, flight controls
- `station.rs` — Station definitions (Helm, Nav, Sensors, Engineering, Engine Room)

### Interactable System

#### Types

**Lever** — analog control, click-and-drag
- State: `position: f32` (0.0 = bottom, 1.0 = top)
- Visual: rod mesh that physically moves along a track
- Interaction: click and hold, drag mouse Y to move. Release to leave at position.
- Mesh: generated by sa_meshgen (cylinder rod + box track + box base)

**Button** — discrete, click
- State: `pressed: bool`, mode: `Momentary` or `Toggle`
- Visual: raised circle/box that depresses when pressed
- Interaction: click to press. Momentary = active while held. Toggle = click on/off.
- Mesh: box on a base plate

**Switch** — multi-position, click to cycle
- State: `position: u8`, `num_positions: u8`
- Visual: handle that flips between positions
- Interaction: click to advance to next position (wraps)
- Mesh: box handle on a pivot

**Screen** — display only
- State: `text_lines: Vec<String>` (updated each frame by ship systems)
- Visual: flat colored panel (screen color from palette)
- No interaction — just shows information
- Mesh: flat box with screen-colored face

#### Interaction Flow

1. Each frame, raycast from camera center forward (1.5m range)
2. If ray hits an interactable entity's collider → set `hovered: Option<Entity>`
3. On mouse click (left button down):
   - If hovered is a Lever → enter drag mode. Store initial mouse Y.
   - If hovered is a Button → set pressed = true
   - If hovered is a Switch → advance position
   - If hovered is a Helm seat → enter seated mode
4. While dragging (mouse held):
   - Track mouse Y delta → map to lever position change
   - Clamp lever position to 0.0..1.0
   - Update lever mesh transform to reflect position
5. On mouse release:
   - Exit drag mode. Lever stays at current position.
   - Button (momentary): set pressed = false

#### Interactable Registration

Each interactable is an ECS entity with:
- `Transform` component (position in ship local space)
- `Interactable` component (type, state, callback ID)
- A physics collider (box) for raycast hit detection
- A mesh handle for rendering

### Helm Seated Mode

When player clicks the helm seat:
1. Camera position locks to the helm viewpoint (fixed position in cockpit, facing forward)
2. Player physics body is disabled (no gravity/collision while seated)
3. Input mode changes:
   - Mouse: pitch (Y) and yaw (X) of the ship
   - W/S: forward/backward thrust (mapped to throttle, not instant)
   - A/D: lateral thrust (RCS)
   - Q/E: roll
   - Space: thrust up, Shift: thrust down
4. Click (left button) to stand up from helm → restore player physics, camera follows player again

While seated, the camera is attached to the ship — as the ship rotates, the view rotates with it.

### Ship Entity

```rust
pub struct Ship {
    pub body_handle: RigidBodyHandle,
    pub mesh_handle: Handle<MeshMarker>,
    pub throttle: f32,           // 0.0..1.0, set by thrust lever
    pub engine_on: bool,         // set by engine button
    pub max_thrust: f32,         // Newtons
    pub max_rcs_thrust: f32,     // Newtons (lateral/vertical)
    pub max_torque: f32,         // Nm (rotation)
}
```

Each frame when helm is active:
- Forward thrust = ship_forward * throttle * max_thrust * engine_on
- Lateral/vertical from WASD/Space/Shift * max_rcs_thrust
- Rotation from mouse + Q/E * max_torque
- Apply forces to physics body

Ship physics values:
- Mass: 50,000 kg
- Max thrust: 500,000 N (10 m/s² acceleration)
- Max RCS: 50,000 N (1 m/s² lateral)
- Max torque: 100,000 Nm
- No drag (Newtonian)

### Initial Ship Setup

The game starts with:
- Ship spawned at WorldPos origin (in space, no ground plane)
- Player spawns inside the ship cockpit
- Ship has gravity inside (mag-boots, simulated as downward force in ship reference frame)

Interactables placed in the ship:
- **Helm seat** — cockpit, centered
- **Thrust lever** — cockpit, right side of helm
- **Engine on/off button** — cockpit, left of helm
- **Speed screen** — cockpit, above helm (shows current velocity)

Future phases add more interactables to other stations (nav console, engineering panel, etc.).

### Meshes (sa_meshgen)

New mesh generators for interactables:

```
lever_mesh(height, travel) -> Mesh       // cylinder rod + box track
button_mesh(size) -> Mesh                // box on base plate
switch_mesh(size, positions) -> Mesh     // handle on pivot
screen_mesh(width, height) -> Mesh       // flat panel
helm_seat_mesh() -> Mesh                 // chair shape
```

These are placed inside the ship at specific positions defined in ship setup code.

### Scene Change

Phase 5a changes the game scene:
- Remove ground plane and test cubes
- Ship mesh is the main geometry
- Player spawns inside the ship
- Ship is in space (star field visible through cockpit window)
- Fly mode (F key) still works for debugging/external view
- Key 7 no longer needed (ship is always visible)
- Key 0 returns to external free camera

## Testing Strategy

- Interactable state: lever moves between 0-1, button toggles, switch cycles
- Ship physics: thrust applies force, no drag, momentum persists
- Helm mode: entering/exiting preserves player state
- Raycast: hits correct interactable, respects range
- Integration: lever position drives thrust, button enables engine
